## 命名规范

NAME.1 各种命名中只支持字符数字和下划线
NAME.2 名字以字母开头，不能以数字和下划线开头
NAME.3 名字中的多个单词通过下划线区分
NAME.4 parameter和define使用大写字母
NAME.5 FSM状态机命名cur_state,next_state
NAME.6 instance名为u_module
NAME.7 总线位宽顺序始终为bus\[高位：低位\]
NAME.8 使用统一的简写表

## 注释

COM.1 输入输出接口说明
COM.2 内部信号说明
COM.3 块功能说明
COM.4 说明简洁，占一行即可

## Coding Style

CS.1 代码中的缩进使用空格，不能使用tab
CS.2 每行代码最多包含一个信号的赋值
CS.3 每行代码最多包含一个输入输出信号的定义
CS.4 所有的内部信号定义放在一起
CS.5 模块实例化时，不能使用位置进行关联
CS.6 端口连接时不能使用表达式
CS.7 状态编码使用parameter进行定义
CS.8 复杂的逻辑关系中使用括号进行优先级管理
CS.9 wire必须被显示定义
CS.10 表达式中相关操作数位宽要保持一致
CS.11 赋值和条件判断时注明数据位宽
CS.12 条件语句中的条件中不直接使用多bit数据作为条件
CS.13 状态机使用三段式风格
CS.14 FSM状态机需设计为主状态机+子状态机的模式
CS.15 always块中只进行一个变量的赋值
CS.16 不能在多个always块中对同一给变量进行多次赋值
CS.17 参数化模块设计
CS.18 parameter, define使用大写
CS.19 数值部分使用十进制,控制使用二进制,编码根据需要
CS.20 数值0不做位宽限定
CS.21 复用重复的组合逻辑
CS.22 逻辑操作和按位逻辑操作不混用
CS.23 赋值语句两边位宽保持一致,进行部分操作时需明确位宽
CS.24 总线定义使用little ending方式进行定义

## 代码赋值规范

SIM.1 时序电路always块中使用<=进行赋值
SIM.2 组合逻辑always块中使用=进行赋值
SIM.3 组合逻辑always敏感列表使用*
SIM.4 时序电路需要显式初始化
SIM.5 不使用x进行赋值
SIM.6 禁止在RTL代码中使用#delay
